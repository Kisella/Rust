# 所有权
所有权是Rust实现内存安全的核心机制，它有一个基本原则:
- 一块内存必须被唯一的变量所拥有
## 1 值语义
数据存放在**栈区**，赋值时发生按位拷贝，拷贝后新对象与原对象相互独立，称为**值语义**。值语义类型发生赋值后将生成一份新的数据，对新数据的操作不会影响原数据，反之亦然。
### 1.1 栈区拷贝
- 值语义类型，数据存放在**栈区**
- 赋值时发生栈区**拷贝**
- 按位拷贝后，新对象与原对象相互独立，不相关
```rust
fn main() {
    let x: i32 = 10;
    let y: i32 = x; //  发生值语义赋值，x 和 y 各自持有一份数据，可以被同时访问
    println!("{x} {y}");
}
```
&nbsp;
### 1.2 Copy Trait
在Rust中，一个类型是否属于值语义，取决于类型是否实现了`Copy`特质。
- 以下类型在标准库中已经实现了`Copy`特质

| 值语义类型 | 举例 |
|---|---|
|数值标量类型|`i32`<br>`f64`<br>`usize`<br>`bool`<br>`char`<br>...|
|共享引用与可变引用|`&i32`<br>`&mut String`|
|裸指针|-|
|所有元素都实现了`Copy`特质的元组|`(1, 3.14)`|
|所有元素都实现了`Copy`特质的数组|`[1, 3.14]`|
|显式实现Copy特质的结构体|`#[derive(Clone, Copy)]`<br>`struct Val { x: i32, y: f64, }`|
|显式实现Copy特质的枚举|`#[derive(Clone, Copy)]`<br>`enum YesOrNo Val { YES, NO, }`|

&nbsp;
- 结构体不会默认实现`Copy`特质，即使它的所有元素都实现了`Copy`特质
- 使用属性宏`#[derive(Clone, Copy)]`为结构体或枚举显式实现Copy特质
```rust
    struct Val {
        x: i32,
        y: f64,
    }

    let v1 = Val{x:1, y:3.14};      //  发生移动
    let v2 = v1;
    println!("{}, {}", v1.x, v1.y);     // 拒绝，已发生移动
    println!("{}, {}", v2.x, v2.y);
```
```rust
    #[derive(Clone, Copy)]  //  显式为结构体实现Copy
    struct Val {
        x: i32,
        y: f64,
    }

    let v1 = Val{x:1, y:3.14};      //  发生拷贝
    let v2 = v1;
    println!("{}, {}", v1.x, v1.y);     // 接受
    println!("{}, {}", v2.x, v2.y);
```

## 2 引用语义
数据存放在**堆区**，使用栈区指针间接访问数据，称为**引用语义**。引用语句类型在赋值时将发生栈区指针拷贝，而堆区数据不变，称为**浅拷贝**。
以下类型是引用语义的：

| 引用语义类型 | 举例 |
| --- | ---- |
| 复合类型 |`String`<br>`Vector`<br>`HashMap`<br>...         |
|元素中含有引用语义类型的的元组|`(1, String::from("Hello"))`|
|元素中含有引用语义类型的数组|`[1, String::from("Hello")]`|
| 元素中含有引用语义类型的结构体     | `-` |
| 变体数据中含有引用语义类型的枚举   | `-` |

&nbsp;
### 2.1 移动
浅拷贝会导致多个指针指向同一块数据，这会带来很多内存安全问题，例如重复释放、垂悬指针、数据竞争等。在Rust中，引用语义赋值是不会发现简单的浅拷贝，而是会发生**所有权移动**来确保**一块内存必须被唯一的变量所拥有**这一所有权规则。
- 移动 = 栈区指针拷贝 + 原变量标记为`moved`
- 被标记为`moved`的变量不能再被使用
```rust
    let s1 = String::from("hello");   //  引用语义类型
    let s2 = s1;    //  发生移动，s1被标记为movd
    println!("{s1}, {s2}");  // 拒绝，s1 不能再使用
```
&nbsp;
### 2.2 深拷贝
- 实现了`Clone`特质的引用语义类型支持进行深拷贝，生成一份新的数据
- 引用语义类型调用`clone()`方法进行深拷贝
- 深拷贝时，栈区指针和堆数据会同时发生拷贝，新的指针指向新的内存
- 深拷贝后的新数据与原数据完全独立，不违反所有权规则
```rust
    let s1 = String::from("hello");  
    let s2 = s1.clone();    //  深拷贝
    println!("{s1}, {s2}");  // 接受，访问的是两份独立的数据
```

## 3 位置表达式与值表达式
位置表达式表示一个内存位置，又称为**左值**。典型的左值有：
- 局部/静态变量 `x`
- 解引用 `*ref`
- 数组索引 `array[index]`
- 字段访问 `expr.f`
- 用括号包裹的左值`(x)` 
&nbsp;

值表达式表示一个实际的值，又称为**右值**。除位置表达式以外的所有形式的表达式都是值表达式。
&nbsp;
## 4 位置上下文与值上下文
需要你提供一个内存位置的场景，称为**位置上下文**。
- 赋值语句左边 `arr[0] = ...`
- 借用/可变借用,`&`符号的右边 `... = &arr[0]` / `... = &mut arr[0]`
- 借用/可变借用,带有`ref`的赋值语句右边 `ref ... = arr[0]` / `ref mut ... = arr[0]`
&nbsp;

需要你提供一个值的场景，称为**值上下文**。一般来说，除位置上下文以外的所有场景都是值上下文。
- 模式匹配中的待匹配表达式（没有`&`或`ref`）
- 表达式求值 `foo(arr[0] + 1)`
- 表达式语句 `foo(arr[0] + 1);`
&nbsp;
## 5 所有权移动的时机
右值通常放在值上下文；左值放在位置上下文时提供内存位置，放在值上下文则提供该位置上的值。当位置表达式出现在值上下文时，就可能出现所有权的移动。即，**在一个需要右值的地方提供一个左值，这个左值就可能发生移动**。
- 在一个需要右值的地方提供左值
- 左值是否实现了`Copy`特质
- 实现`Copy`特质 -> 栈区拷贝
- 未实现`Copy`特质 -> 发生移动
```rust
    let s1 = String::from("hello");
    let s2 = s1;  // 值上下文 + 位置表达式 + 未实现Copy特质 -> 移动

    let s1 = String::from("hello");
    s1;     //  表达式语句，会发生移动，值直接被丢弃，不推荐。推荐使用drop(s1);
```
&nbsp;
## 6 权限
当一个变量拥有某块内存的所有权时，它可以：
- 释放资源(变量生命周期结束)
- 移动所有权(Heap Only)
- 借用(声明共享引用或可变引用)
&nbsp;
### 6.1 释放资源
#### 6.1.1 RAII 机制
当一个变量的生命周期结束时，它拥有的所有资源将被释放。这种将“资源的生命周期”与“变量的生命周期”绑定在一起的机制，称为**RAII**，即资源获取即初始化 Resource Acquisition Is Initialization.
- 值语义 -> 栈帧自动销毁，无需特殊处理
- 引用语义 -> 需要额外处理堆上的资源释放 -> 通过`Drop`特质
&nbsp;
#### 6.1.2 `Drop`特质
`Drop`是一个特殊的Trait，用于引用语义的变量在生命周期结束时执行清理逻辑。Rust在值离开作用域时自动调用drop, 通常无需手动调用。
- 当拥有所有权的变量离开作用域时会调用drop方法
- 变量离开作用域时，Rust会递归式触发自己及其所有子变量的drop方法
- Rust不允许同一类型同时实现`Copy`和`Drop`特质
```rust
#[derive(Debug, Default)]
struct Val {
    x: i32,
    y: f64,
}

impl Drop for Val {
    fn drop(&mut self) {
        println!("Dropping Val: {:?}", self);
    }
}

fn main() {
    let v1 = Val::default();
    let v2 = Val { x: 20, y: 2.71 };
    let v3 = Val { x: 30, y: 1.61 };
    println!("v1: {:?}", v1);
    println!("v2: {:?}", v2);
    // 显式调用drop函数
    drop(v1); // v1被丢弃，调用drop方法
    println!("After dropping v1");
    {
        let v = v2; // v2被移动到v中，v2不再可用
    }   //  v生命周期结束，调用drop方法
    // v3会在main函数结束时自动被丢弃，调用drop方法
}
```
&nbsp;
### 6.2 借用
- 只有拥有所有权的变量才能声明共享引用或可变引用
```rust
    let s1 = String::from("hello");
    let s2 = s1;

    // let r1 = &s1; // s1被标记为moved, 不能借用s1
    let r2 = &s2; // s2拥有所有权，可以借用s2
    println!("{r2}");
```
&nbsp;
- 变量被移动或释放时，它的所有引用失效
```rust
    let s1 = String::from("hello");
    let s2 = String::from("world");
    let r1 = &s1;
    let r2 = &s2;
    println!("{r1} {r2}");

    let s3 = s1;
    // println!("{r1}"); // r1借用s1, s1被移动, r1不再有效
    drop(s2);
    // println!("{r2}"); // r2借用s2, s2被释放, r2不再有效
```
&nbsp;
## 7 小结
- `=`符号左边和`&`符号右边的位置为位置上下文，除此之外的都基本为值上下文
- 在一个需要右值的地方输入一个左值，且该左值未实现Copy特质时，将发生移动
- 移动 = 栈区指针拷贝 + 原变量被标记为"moved"
- 只有拥有所有权的变量才能被借用、释放或移动
